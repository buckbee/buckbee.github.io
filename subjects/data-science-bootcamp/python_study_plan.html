Study Plan Overview

Master Python fundamentals essential for data science roles. This 8-week plan covers pandas, NumPy, code quality, performance optimization, and professional development practices used in production environments.

What You'll Learn

  • Advanced pandas operations for data manipulation and analysis
  • NumPy fundamentals including array operations and broadcasting
  • Professional code quality practices and project structure
  • Performance optimization techniques for large datasets
  • Environment management and development tools

Prerequisites

  • Basic Python syntax and programming concepts
  • Understanding of data structures (lists, dictionaries)
  • Familiarity with command line operations

Learning Resources

  • Book: "Python for Data Analysis" by Wes McKinney
  • Course: DataCamp's "Pandas for Data Science"
  • Documentation: Official pandas and NumPy documentation
  • Practice: Kaggle datasets and notebooks

Weekly Structure

Each week includes:

  • Conceptual learning (2-3 hours)
  • Hands-on coding exercises (3-4 hours)
  • Practice projects (2-3 hours)
  • Review and consolidation (1-2 hours)

Practice Exercise:

from typing import List, Dict
import pandas as pd

def process_data(df: pd.DataFrame, 
                 columns: List[str]) -> pd.DataFrame:
    """
    Process DataFrame by selecting and cleaning columns.
    
    Args:
        df: Input DataFrame
        columns: List of column names to keep
        
    Returns:
        Processed DataFrame with selected columns
        
    Raises:
        ValueError: If columns don't exist in DataFrame
    """
    if not all(col in df.columns for col in columns):
        raise ValueError("Invalid columns specified")
    
    return df[columns].dropna()

Week 6: Performance Optimization

Learning Objectives

  • Profile and identify performance bottlenecks
  • Optimize pandas operations
  • Use vectorization effectively
  • Implement parallel processing

Topics to Cover

1. Profiling Code

import time
from line_profiler import LineProfiler

# Simple timing
start = time.time()
# your code here
print(f"Execution time: {time.time() - start:.2f}s")

# Detailed profiling with line_profiler
profiler = LineProfiler()
profiler.add_function(your_function)
profiler.run('your_function()')

2. Pandas Optimization

  • Use vectorized operations instead of loops
  • Choose appropriate data types
  • Use apply() wisely vs vectorization
  • Chunk large files for processing

Practice Exercise:

# BAD: Using iterrows (slow)
for idx, row in df.iterrows():
    df.at[idx, 'new_col'] = row['col1'] * row['col2']

# GOOD: Vectorized operation (fast)
df['new_col'] = df['col1'] * df['col2']

# Optimize data types
df['category'] = df['category'].astype('category')
df['amount'] = df['amount'].astype('float32')

3. Working with Large Datasets

# Read in chunks
chunk_size = 10000
chunks = []
for chunk in pd.read_csv('large_file.csv', chunksize=chunk_size):
    processed = process_chunk(chunk)
    chunks.append(processed)
result = pd.concat(chunks)

# Use Dask for larger-than-memory datasets
import dask.dataframe as dd
ddf = dd.read_csv('huge_file.csv')
result = ddf.groupby('category')['value'].mean().compute()
