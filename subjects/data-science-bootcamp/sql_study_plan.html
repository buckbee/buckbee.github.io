Study Plan Overview

Master SQL from complex queries to query optimization. This 6-week plan covers everything from window functions and CTEs to performance tuning and advanced analytics needed for data science roles.

What You'll Learn

  • Complex queries with multiple joins and subqueries
  • Window functions for advanced analytics
  • Query optimization and performance tuning
  • Data manipulation (INSERT, UPDATE, DELETE)
  • Advanced aggregations and pivoting
  • Working with real-world data warehouses

Prerequisites

  • Basic SQL knowledge (SELECT, WHERE, JOIN)
  • Understanding of relational database concepts
  • Familiarity with basic aggregations (COUNT, SUM, AVG)

Learning Resources

  • Practice Platforms: LeetCode SQL, HackerRank, DataLemur
  • Books: "SQL Performance Explained" by Markus Winand
  • Documentation: PostgreSQL, MySQL, SQL Server docs
  • Courses: Mode Analytics SQL Tutorial, Khan Academy

Database Setup

For practice, you'll need access to a SQL database. Options include:

  • Local: Install PostgreSQL or MySQL locally
  • Cloud: Use SQLite online, db-fiddle.com, or SQL Fiddle
  • Sample Databases: Download Northwind, Sakila, or AdventureWorks

Weekly Time Commitment

  • Conceptual learning: 2-3 hours
  • Practice problems: 4-5 hours
  • Projects: 2-3 hours
  • Total: 8-11 hours per week

INSERT Operations

Topics to Cover:

  • Single and multiple row inserts
  • INSERT with subqueries
  • RETURNING clause
  • Bulk inserts from CSV/file

Practice Examples:

-- Single row insert
INSERT INTO customers (customer_name, email, country)
VALUES ('John Doe', 'john@example.com', 'Finland');

-- Multiple rows
INSERT INTO products (product_name, price, category)
VALUES 
    ('Laptop', 999.99, 'Electronics'),
    ('Mouse', 29.99, 'Electronics'),
    ('Keyboard', 79.99, 'Electronics');

-- Insert with subquery
INSERT INTO archived_orders (order_id, customer_id, order_date, total)
SELECT order_id, customer_id, order_date, order_total
FROM orders
WHERE order_date < '2023-01-01';

-- Insert with RETURNING (get inserted ID)
INSERT INTO customers (customer_name, email)
VALUES ('Jane Smith', 'jane@example.com')
RETURNING customer_id, created_at;

UPDATE Operations

Topics to Cover:

  • Conditional updates with WHERE
  • UPDATE with JOIN (FROM clause)
  • UPDATE with CASE statements
  • Bulk updates

Practice Examples:

-- Basic update
UPDATE products
SET price = price * 1.1
WHERE category = 'Electronics';

-- Update with JOIN
UPDATE orders o
SET status = 'Shipped'
FROM customers c
WHERE o.customer_id = c.customer_id
  AND c.country = 'Finland'
  AND o.order_date > '2024-01-01';

-- Conditional update with CASE
UPDATE employees
SET salary = CASE
    WHEN performance_rating = 'Excellent' THEN salary * 1.15
    WHEN performance_rating = 'Good' THEN salary * 1.10
    WHEN performance_rating = 'Average' THEN salary * 1.05
    ELSE salary
END
WHERE last_review_date >= '2024-01-01';

-- Update with subquery
UPDATE products
SET average_rating = (
    SELECT AVG(rating)
    FROM reviews
    WHERE reviews.product_id = products.product_id
);

DELETE Operations

Topics to Cover:

  • Safe deletion practices
  • DELETE with JOIN
  • CASCADE deletes
  • Soft deletes vs hard deletes

Practice Examples:

-- Basic delete
DELETE FROM orders
WHERE order_date < '2020-01-01';

-- Delete with subquery
DELETE FROM customers
WHERE customer_id IN (
    SELECT customer_id
    FROM orders
    GROUP BY customer_id
    HAVING MAX(order_date) < '2022-01-01'
);

-- Delete with JOIN (PostgreSQL)
DELETE FROM order_items oi
USING orders o
WHERE oi.order_id = o.order_id
  AND o.status = 'Cancelled';

-- Soft delete (preferred for data retention)
UPDATE customers
SET deleted_at = CURRENT_TIMESTAMP,
    status = 'Inactive'
WHERE customer_id = 12345;

Transactions & ACID Properties

Topics to Cover:

  • BEGIN, COMMIT, ROLLBACK
  • SAVEPOINT for partial rollbacks
  • Transaction isolation levels
  • Handling concurrent updates

Practice Examples:

-- Basic transaction
BEGIN;

UPDATE accounts 
SET balance = balance - 100 
WHERE account_id = 1;

UPDATE accounts 
SET balance = balance + 100 
WHERE account_id = 2;

COMMIT;

-- Transaction with SAVEPOINT
BEGIN;

INSERT INTO orders (customer_id, order_total) 
VALUES (123, 500.00);

SAVEPOINT before_items;

INSERT INTO order_items (order_id, product_id, quantity)
VALUES (LASTVAL(), 456, 2);

-- If something goes wrong with items
ROLLBACK TO SAVEPOINT before_items;

-- Continue with other operations
COMMIT;

UPSERT (INSERT ... ON CONFLICT)

Practice Examples:

-- Insert or update if exists
INSERT INTO user_stats (user_id, login_count, last_login)
VALUES (123, 1, CURRENT_TIMESTAMP)
ON CONFLICT (user_id) 
DO UPDATE SET 
    login_count = user_stats.login_count + 1,
    last_login = EXCLUDED.last_login;

-- Insert or do nothing
INSERT INTO unique_visitors (user_id, visit_date)
VALUES (456, '2024-01-15')
ON CONFLICT (user_id, visit_date) DO NOTHING;

Date/Time Analytics

Topics to Cover:

  • Date extraction and formatting
  • Date arithmetic and intervals
  • Time series analysis
  • Handling time zones

Practice Examples:

-- Date extraction
SELECT 
    order_date,
    EXTRACT(YEAR FROM order_date) AS year,
    EXTRACT(MONTH FROM order_date) AS month,
    EXTRACT(DOW FROM order_date) AS day_of_week,
    DATE_TRUNC('week', order_date) AS week_start
FROM orders;

-- Date arithmetic
SELECT 
    customer_id,
    first_order_date,
    last_order_date,
    AGE(last_order_date, first_order_date) AS customer_lifetime,
    DATE_PART('day', last_order_date - first_order_date) AS days_active
FROM customer_summary;

-- Generate date series
SELECT 
    generate_series(
        '2024-01-01'::date,
        '2024-12-31'::date,
        '1 day'::interval
    )::date AS date;

Cohort Analysis

Practice Examples:

-- Monthly cohort retention
WITH cohorts AS (
    SELECT 
        customer_id,
        DATE_TRUNC('month', MIN(order_date)) AS cohort_month
    FROM orders
    GROUP BY customer_id
),
cohort_activity AS (
    SELECT 
        c.cohort_month,
        DATE_TRUNC('month', o.order_date) AS activity_month,
        COUNT(DISTINCT o.customer_id) AS active_customers
    FROM cohorts c
    JOIN orders o ON c.customer_id = o.customer_id
    GROUP BY c.cohort_month, DATE_TRUNC('month', o.order_date)
)
SELECT 
    cohort_month,
    activity_month,
    active_customers,
    EXTRACT(MONTH FROM AGE(activity_month, cohort_month)) AS month_number
FROM cohort_activity
ORDER BY cohort_month, activity_month;

Customer Analytics

Key Metrics:

  • Customer Lifetime Value (LTV)
  • Churn rate
  • Average order value
  • Purchase frequency
  • RFM analysis (Recency, Frequency, Monetary)

Practice Examples:

-- Customer Lifetime Value
SELECT 
    customer_id,
    COUNT(*) AS order_count,
    SUM(order_total) AS total_spent,
    AVG(order_total) AS avg_order_value,
    MAX(order_date) AS last_order_date,
    DATE_PART('day', CURRENT_DATE - MAX(order_date)) AS days_since_last_order
FROM orders
GROUP BY customer_id;

-- RFM Analysis
WITH rfm AS (
    SELECT 
        customer_id,
        DATE_PART('day', CURRENT_DATE - MAX(order_date)) AS recency,
        COUNT(*) AS frequency,
        SUM(order_total) AS monetary
    FROM orders
    GROUP BY customer_id
)
SELECT 
    customer_id,
    recency,
    frequency,
    monetary,
    NTILE(5) OVER (ORDER BY recency DESC) AS r_score,
    NTILE(5) OVER (ORDER BY frequency ASC) AS f_score,
    NTILE(5) OVER (ORDER BY monetary ASC) AS m_score
FROM rfm;

-- Churn analysis
WITH customer_activity AS (
    SELECT 
        customer_id,
        MAX(order_date) AS last_order_date,
        COUNT(*) AS total_orders
    FROM orders
    GROUP BY customer_id
)
SELECT 
    COUNT(*) AS total_customers,
    COUNT(*) FILTER (WHERE last_order_date < CURRENT_DATE - INTERVAL '90 days') AS churned_customers,
    ROUND(
        100.0 * COUNT(*) FILTER (WHERE last_order_date < CURRENT_DATE - INTERVAL '90 days') / COUNT(*),
        2
    ) AS churn_rate_pct
FROM customer_activity
WHERE total_orders > 1;

Working with JSON

Topics to Cover:

  • JSON operators (->, ->>, #>)
  • Extracting JSON fields
  • Filtering on JSON data
  • Aggregating JSON arrays

Practice Examples:

-- Extract JSON fields
SELECT 
    id,
    metadata->>'customer_name' AS customer_name,
    metadata->>'email' AS email,
    (metadata->>'age')::int AS age
FROM user_profiles
WHERE metadata->>'country' = 'Finland';

-- Query nested JSON
SELECT 
    order_id,
    jsonb_array_elements(items) AS item,
    jsonb_array_elements(items)->>'product_id' AS product_id,
    (jsonb_array_elements(items)->>'quantity')::int AS quantity
FROM orders_json;

-- JSON aggregation
SELECT 
    category,
    json_agg(
        json_build_object(
            'product_name', product_name,
            'price', price
        )
    ) AS products
FROM products
GROUP BY category;

Data Warehouse Concepts

Topics to Cover:

  • Star schema vs snowflake schema
  • Fact tables and dimension tables
  • Slowly Changing Dimensions (SCD)

Example Fact Table Query:

-- Sales fact table with dimensional joins
SELECT 
    d.date,
    d.year,
    d.quarter,
    d.month,
    p.product_name,
    p.category,
    c.customer_name,
    c.segment,
    s.region,
    f.quantity,
    f.revenue,
    f.cost,
    f.revenue - f.cost AS profit
FROM fact_sales f
JOIN dim_date d ON f.date_key = d.date_key
JOIN dim_product p ON f.product_key = p.product_key
JOIN dim_customer c ON f.customer_key = c.customer_key
JOIN dim_store s ON f.store_key = s.store_key
WHERE d.year = 2024
  AND p.category = 'Electronics';

SELECT

    p.product_name,
    od.quantity,

SELECT customer_name

    Day 4-7: Common Table Expressions (CTEs)

    Topics to Cover:

    • Creating and using CTEs
    • Multiple CTEs in one query
    • Recursive CTEs for hierarchical data
    • When to use CTEs vs subqueries

    Practice Examples:

    -- Basic CTE
WITH high_value_customers AS (
    SELECT 
        customer_id,
        SUM(order_total) AS total_spent
    FROM orders
    GROUP BY customer_id
    HAVING SUM(order_total) > 10000
)
SELECT 
    c.customer_name,
    h.total_spent
FROM customers c
INNER JOIN high_value_customers h ON c.customer_id = h.customer_id;

-- Multiple CTEs
WITH 
monthly_sales AS (
    SELECT 
        DATE_TRUNC('month', order_date) AS month,
        SUM(order_total) AS total_sales
    FROM orders
    GROUP BY DATE_TRUNC('month', order_date)
),
avg_monthly_sales AS (
    SELECT AVG(total_sales) AS avg_sales
    FROM monthly_sales
)
SELECT 
    m.month,
    m.total_sales,
    a.avg_sales,
    m.total_sales - a.avg_sales AS variance
FROM monthly_sales m
CROSS JOIN avg_monthly_sales a
ORDER BY m.month;

    Week 2: Window Functions

    Topics to Cover:

    • ROW_NUMBER(), RANK(), DENSE_RANK()
    • LAG() and LEAD() for comparing rows
    • Running totals with SUM() OVER()
    • Moving averages
    • PARTITION BY for grouped calculations

    Practice Examples:

    -- Ranking
SELECT 
    employee_name,
    department,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS salary_rank
FROM employees;

-- Running total
SELECT 
    order_date,
    order_total,
    SUM(order_total) OVER (ORDER BY order_date) AS running_total
FROM orders;

-- LAG and LEAD (comparing to previous/next row)
SELECT 
    order_date,
    order_total,
    LAG(order_total) OVER (ORDER BY order_date) AS previous_order,
    LEAD(order_total) OVER (ORDER BY order_date) AS next_order,
    order_total - LAG(order_total) OVER (ORDER BY order_date) AS difference
FROM orders;

-- Moving average
SELECT 
    order_date,
    order_total,
    AVG(order_total) OVER (
        ORDER BY order_date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) AS moving_avg_7days
FROM orders;

    Advanced GROUP BY

    Topics to Cover:

    • Multiple column grouping
    • HAVING vs WHERE clauses
    • Conditional aggregations with CASE
    • FILTER clause (PostgreSQL)

    Practice Examples:

    -- Multiple aggregations with CASE
SELECT 
    product_category,
    COUNT(*) AS total_orders,
    SUM(order_total) AS total_revenue,
    AVG(order_total) AS avg_order_value,
    SUM(CASE WHEN order_total > 100 THEN 1 ELSE 0 END) AS high_value_orders,
    SUM(CASE WHEN order_total > 100 THEN order_total ELSE 0 END) AS high_value_revenue
FROM orders o
JOIN products p ON o.product_id = p.product_id
GROUP BY product_category
HAVING SUM(order_total) > 10000;

-- Conditional aggregation
SELECT 
    DATE_TRUNC('month', order_date) AS month,
    COUNT(*) AS total_orders,
    COUNT(*) FILTER (WHERE order_total > 50) AS orders_over_50,
    SUM(order_total) AS total_revenue,
    SUM(order_total) FILTER (WHERE customer_type = 'Premium') AS premium_revenue
FROM orders
GROUP BY DATE_TRUNC('month', order_date);

    ROLLUP and CUBE

    Topics to Cover:

    • Creating subtotals with ROLLUP
    • Multi-dimensional aggregations with CUBE
    • GROUPING SETS for custom groupings

    Practice Examples:

    -- ROLLUP for hierarchical subtotals
SELECT 
    region,
    country,
    city,
    SUM(sales) AS total_sales
FROM sales_data
GROUP BY ROLLUP (region, country, city)
ORDER BY region, country, city;

-- CUBE for all possible combinations
SELECT 
    product_category,
    customer_segment,
    SUM(revenue) AS total_revenue
FROM sales
GROUP BY CUBE (product_category, customer_segment);

-- GROUPING SETS for custom groupings
SELECT 
    year,
    quarter,
    month,
    SUM(sales) AS total_sales
FROM sales
GROUP BY GROUPING SETS (
    (year),
    (year, quarter),
    (year, quarter, month)
);

    Pivot Tables in SQL

    Practice Examples:

    -- Creating a pivot table (monthly sales by product)
SELECT 
    product_name,
    SUM(CASE WHEN EXTRACT(MONTH FROM order_date) = 1 THEN sales ELSE 0 END) AS Jan,
    SUM(CASE WHEN EXTRACT(MONTH FROM order_date) = 2 THEN sales ELSE 0 END) AS Feb,
    SUM(CASE WHEN EXTRACT(MONTH FROM order_date) = 3 THEN sales ELSE 0 END) AS Mar,
    SUM(CASE WHEN EXTRACT(MONTH FROM order_date) = 4 THEN sales ELSE 0 END) AS Apr
FROM sales
GROUP BY product_name;

-- Using CROSSTAB (PostgreSQL)
SELECT *
FROM CROSSTAB(
    'SELECT product_name, month, sales
     FROM monthly_sales
     ORDER BY 1, 2'
) AS ct(product_name TEXT, Jan NUMERIC, Feb NUMERIC, Mar NUMERIC);

    Topics to Cover:

    • Reading execution plans
    • Identifying bottlenecks (Seq Scan vs Index Scan)

    Practice Examples:

    -- Basic EXPLAIN
EXPLAIN
SELECT * 
FROM orders
WHERE order_date > '2024-01-01';

-- EXPLAIN ANALYZE (actually runs the query)
EXPLAIN ANALYZE
SELECT 
    c.customer_name,
    COUNT(*) AS order_count,
    SUM(o.order_total) AS total_spent
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_name
HAVING SUM(o.order_total) > 1000;

    Indexing Strategies

    Topics to Cover:

    • When to create indexes
    • Single-column vs multi-column indexes
    • Index types (B-tree, Hash, GiST)
    • Index maintenance and monitoring

    Practice Examples:

    -- Create single-column index
CREATE INDEX idx_orders_date ON orders(order_date);

-- Multi-column index
CREATE INDEX idx_orders_customer_date ON orders(customer_id, order_date);

-- Partial index (for specific conditions)
CREATE INDEX idx_active_customers ON customers(customer_id)
WHERE status = 'active';

-- Check index usage
SELECT 
    schemaname,
    tablename,
    indexname,
    idx_scan,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;

    Query Optimization Techniques

    Best Practices:

    • Use LIMIT when testing queries
    • Avoid SELECT * (specify needed columns)
    • Use EXISTS instead of IN for subqueries
    • Optimize JOIN order
    • Use UNION ALL instead of UNION when appropriate
    • Avoid functions in WHERE clauses on indexed columns

    Before and After Examples:

    -- SLOW: Function on indexed column
SELECT * FROM orders
WHERE DATE(order_timestamp) = '2024-01-15';

-- FAST: Use range query
SELECT * FROM orders
WHERE order_timestamp >= '2024-01-15'
  AND order_timestamp < '2024-01-16';

-- SLOW: Using IN with large subquery
SELECT * FROM customers
WHERE customer_id IN (
    SELECT customer_id FROM orders WHERE order_total > 100
);

-- FAST: Using EXISTS
SELECT * FROM customers c
WHERE EXISTS (
    SELECT 1 FROM orders o 
    WHERE o.customer_id = c.customer_id 
    AND o.order_total > 100
);

-- SLOW: Multiple OR conditions
SELECT * FROM products
WHERE category = 'Electronics'
   OR category = 'Computers'
   OR category = 'Phones';

-- FAST: Use IN
SELECT * FROM products
WHERE category IN ('Electronics', 'Computers', 'Phones');
